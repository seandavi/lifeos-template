# Examples

Real LifeOS files don't look like the empty templates do. These are filled-in samples so you can see what good looks like before you start.

Everything here is fictional. The persona is **Pat Taylor, engineering team lead** at a mid-sized SaaS company — six direct reports, owns a platform team, reports to a VP, doesn't ship code daily but stays close to architecture. The work is roadmap planning, mentoring, oncall, hiring, and meetings. All names are placeholders (Sally Sanders, John Smith, etc.) — substitute your own context; the shapes transfer.

## Files

| File | What it shows |
|------|---------------|
| [`now.md`](now.md) | A filled-in current-state snapshot. Could brief a colleague on what you're doing in 60 seconds. |
| [`inbox.md`](inbox.md) | Action-needed / waiting-on / keep-warm / parked — what an inbox looks like mid-week. |
| [`completed.md`](completed.md) | Two months of completed items with strategic context. The "impact" line is what makes this useful at performance review. |
| [`journal-day.md`](journal-day.md) | A single substantive daily entry — Franklin frame, calendar pulled, log entries, evening reflection. |
| [`journal-week-review.md`](journal-week-review.md) | A `/weekly-review` output. State, wins, drift, dropped, carry-forward, one forced question. |
| [`decision-prospective.md`](decision-prospective.md) | A `/decide` capture — alternatives, prediction at 30/90/365, confidence, worst case, revisit date. |
| [`decision-revisited.md`](decision-revisited.md) | The same decision three months later with a revisit block appended. Prediction vs reality. This is the loop most templates skip. |
| [`note.md`](note.md) | An atomic note — one idea, heavy on wiki-links, no preamble. |
| [`person.md`](person.md) | A relationship note used by `/meeting-prep`. |

## How to read these

- **Notice the wiki-links.** `[[sally-sanders]]`, `[[platform-migration]]`, `[[meeting-restructure]]` — every example points at others. That's how the system stays connected.
- **Notice what's not here.** No headers for headers' sake. No checklist filler. Each line earns its place.
- **Notice the tone.** Direct. Names. Numbers. Dates. No hedging. That's the standard the skills push toward.

## Should I copy these into my vault?

No. They're for reading, not for use. Your `now.md` should reflect *your* week, not this one. Empty templates live in [`../templates/`](../templates/).
